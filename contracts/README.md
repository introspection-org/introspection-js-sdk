# Vendored platform contract snapshot

OpenAPI contract documents and wire fixtures vendored from the Introspection
platform monorepo. They are the source of truth the conformance suite in
`tests/contracts/` checks the SDK's types and parsers against.

- **Source repo:** `introspection-cloud`, `contracts/` directory
- **Source branch:** `claude/cli-implementation-review-aqpbx5`
- **`generated_from_commit`:** `0682b980f2835ded036e1eee710dd311b3680984`
  (from `manifest.json`; informational — artifact identity is the content
  sha256 recorded per file in the manifest)

## Contents

| Path                          | What it is                                                                                                  |
| ----------------------------- | ----------------------------------------------------------------------------------------------------------- |
| `manifest.json`               | sha256 per artifact, as generated by the platform's `make contracts`                                        |
| `controlplane.public.v1.json` | Control Plane public `/v1` OpenAPI document                                                                 |
| `dataplane.public.v1.json`    | Data Plane public `/v1` OpenAPI document                                                                    |
| `public-surface.yaml`         | Declared public route surface                                                                               |
| `fixtures/`                   | Canonical wire fixtures (task create, runner spec, pagination envelopes, error envelopes, AG-UI run stream) |

The `*.full.v1.json` documents listed in `manifest.json` are internal-surface
contracts and are deliberately **not** vendored here; the conformance suite
only verifies the artifacts present in this directory.

## Syncing

Sync is **manual** for now: copy the files above from the source repo's
`contracts/` directory and re-run `pnpm test`. The
`tests/contracts/` suite fails if any vendored file's sha256 stops matching
`manifest.json`, so hand-edits to fixtures (instead of a proper re-vendor)
are caught. A `make`/script-driven sync is a follow-up.

Fixtures are the platform's truth — when a fixture disagrees with an SDK
type, the SDK adapts (or the mismatch is reported upstream); do not edit
fixtures to make tests pass.
